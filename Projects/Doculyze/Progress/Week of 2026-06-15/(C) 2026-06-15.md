# (C) 2026-06-15 — Data layer designed (Firestore + ChromaDB two-store split)

## Done
- Designed the **data layer** ([[(C) Data Layer|Data Layer]]) as a **two-store split**: Firestore holds ownership + the Documents-list UI (per-user isolation via security rules); ChromaDB holds the chunk-grained semantic index — no separate Firestore `chunks` collection, a citation is just a retrieved chunk's `docId:chunkIndex`.
- Settled **ChromaDB** for the vector index: free, local-first, no infra to host for the ~2-min local demo; documented its two caveats (no built-in row-level security → always inject `where: { userId }`; weak native hybrid/BM25 → lean improved-RAG story on metadata filtering, re-ranking, HyDE).
- Wrote concrete "configure the database" steps (local Chroma persistent path, single collection + `userId` filter, cosine distance, pin the embedding model/dimension).

## Decisions
- **Decided:** Two-store data layer — Firestore for ownership/list, ChromaDB for the vector index — [[(C) Two-Store Data Layer|Two-Store Data Layer]].
