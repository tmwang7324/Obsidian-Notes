# (C) 2026-06-25 — Two-store data layer decided; DAL + session primitives shipped

**Advances:** [[Doculyze MVP]]

> Work from the night-of-2026-06-24 `/grill-me` session; supporting files landed after midnight (2026-06-25).

## Done
- Resolved **document-storage architecture**: **Firestore** owns doc metadata/refs (replacing MongoDB), **Firebase Storage** holds raw bytes + a canonical `extracted.txt`, **ChromaDB** holds chunk text + vectors, **Redis** is a pure cache (versioned keys, TTL + `allkeys-lru`). Data model: subcollection `users/{uid}/documents/{docId}` with a server-generated id.
- Shipped supporting code: `next.config.ts` `bodySizeLimit → 15mb`; `admin.tsx` now exports `storage`; **new `_lib/session.tsx`** (`getSession`/`getCurrentUid`/`requireUid`, cookie-verified, React-`cache()` deduped); **new `_lib/database.tsx` DAL** (`createDocumentRecord`/`listDocuments`, uid derived internally, auto-id, `status:"uploaded"`, `version:1`); `verify_user.tsx` delegates to `getCurrentUid()`.
- Found bugs in the old `upload_document.tsx` stub: keyed by user-supplied title (collision/overwrite), `verifyUser()` treated as a uid (it's a boolean), and client-SDK `doc()/setDoc()` used against the Admin SDK.
- Learning notes: **Data Access Layer** (page-level auth vs. DAL — why unprotected pages leak nested data components) and **Errors To Take Note Of** (Server-Action `File`→`Blob` serialization drops `.name`; `createRefresh`/`verifyUser` race in `onAuthStateChanged`; `<Header>` must sit inside `<body>` above `{children}`).

## Decisions
- **Decided:** Firestore + Storage + ChromaDB + Redis replace MongoDB for the document layer — [[(C) Two-Store Data Layer|Two-Store Data Layer]]
