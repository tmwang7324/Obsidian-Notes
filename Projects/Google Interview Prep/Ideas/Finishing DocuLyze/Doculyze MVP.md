# Goal
Before the Google Interview, I want to have a MVP of the Doculyze application, so I can showcase RAG/LangChain, Firebase, Next.js, and RabbitMQ expertise.

I want to do this project not only with the goal of deriving exceptional STAR stories from the process and actually knowing what I'm talking about if asked about it. But, also grasp lasting techniques such as model fine-tuning (for LLMs and Transformers), efficient data storage, and microservice architecture.

First, implement a basic naive RAG system. Only include barebones document chunking, similarity score for retrieval, and primitive data labelling.

Improve upon this process with researched RAG improvements, such as Hybrid search, vital metadata labelling, top K re-ranking, Hypothetical Document Embeddings, etc.
